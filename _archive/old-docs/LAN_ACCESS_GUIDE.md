# 局域网访问配置指南

## 🎯 功能说明

本项目已配置为支持局域网访问，您可以在以下设备上测试：
- 📱 **手机**（iOS/Android）
- 💻 **其他电脑**（Windows/Mac/Linux）
- 📟 **平板**（iPad/Android Tablet）

通过 Tailscale 创建的虚拟局域网，您可以从任何地方访问这个项目。

---

## 🚀 快速开始

### 第一步：配置防火墙（首次使用）

**以管理员身份运行** `setup-firewall.bat`：

1. 右键点击 `setup-firewall.bat`
2. 选择 **"以管理员身份运行"**
3. 如果弹出 UAC 提示，点击 **"是"**
4. 等待防火墙规则添加完成

> ⚠️ **重要**：如果不配置防火墙，其他设备可能无法访问！

---

### 第二步：启动局域网服务器

#### 方法 A：Python 版本（推荐）

双击运行 `start-lan-server.bat`

**优点**：
- ✅ Python 通常已预装
- ✅ 无需额外安装依赖
- ✅ 轻量快速

#### 方法 B：Node.js 版本

双击运行 `start-lan-server-node.bat`

**优点**：
- ✅ 自动打开浏览器
- ✅ 更详细的日志输出

---

### 第三步：在其他设备上访问

启动服务器后，会显示类似这样的信息：

```
╔══════════════════════════════════════════════════════════╗
║                                                          ║
║  📍 本地访问: http://localhost:8080                     ║
║  📱 手机访问: http://100.64.123.45:8080                 ║
║  💻 其他电脑: http://100.64.123.45:8080                 ║
║                                                          ║
║  ⚠️  重要提示:                                           ║
║     1. 确保 Tailscale 已连接                             ║
║     2. 防火墙允许 8080 端口                              ║
║     3. 在其他设备上使用上面的 IP 地址访问                 ║
║                                                          ║
╚══════════════════════════════════════════════════════════╝
```

**在手机/其他电脑上**：
1. 确保设备已连接到 Tailscale
2. 打开浏览器（Chrome/Safari/Firefox）
3. 输入显示的 IP 地址，例如：`http://100.64.123.45:8080`
4. 按 Enter 访问

## 🔍 故障排查

### 问题 1：其他设备无法访问

**检查清单**：

1. ✅ **Tailscale 是否连接**
   - 在本机和其他设备上都检查 Tailscale 状态
   - 应该显示 "Connected"

2. ✅ **防火墙是否配置**
   - 运行 `setup-firewall.bat`
   - 或者手动添加防火墙规则（见下方）

3. ✅ **IP 地址是否正确**
   - 在启动脚本中查看显示的 IP 地址
   - 确保在其他设备上输入的是这个 IP

4. ✅ **服务器是否正在运行**
   - 在本机浏览器访问 `http://localhost:8080`
   - 如果能访问，说明服务器正常

5. ✅ **端口是否被占用**
   - 如果 8080 端口被占用，尝试修改端口号
   - 编辑启动脚本，将 `8080` 改为其他端口（如 `8081`）

---

### 问题 2：防火墙配置失败

**手动添加防火墙规则**：

1. 按 `Win + R`，输入 `wf.msc`，按 Enter
2. 点击左侧的 **"入站规则"**
3. 点击右侧的 **"新建规则"**
4. 选择 **"端口"** → 下一步
5. 选择 **"TCP"**，特定本地端口输入 **"8080"** → 下一步
6. 选择 **"允许连接"** → 下一步
7. 勾选所有配置文件（域、专用、公用）→ 下一步
8. 名称输入：**MatSci Calculator Lab - HTTP 8080**
9. 点击 **"完成"**

---

### 问题 3：Tailscale 连接问题

**检查步骤**：

1. 打开 Tailscale 应用
2. 确认状态为 **"Connected"**
3. 查看分配的 IP 地址（通常是 `100.x.x.x` 或 `100.64.x.x`）
4. 在其他设备上也确认 Tailscale 已连接
5. 尝试 ping 对方的 Tailscale IP

**重启 Tailscale**：
- 退出 Tailscale
- 重新启动
- 重新登录

---

### 问题 4：手机浏览器显示异常

**常见问题**：

1. **布局错乱**
   - 这是正常的！这正是我们需要在手机端测试的原因
   - 记录显示问题，回来修复 CSS

2. **小键盘按钮无法点击**
   - 打开浏览器的开发者工具（Chrome: 三点菜单 → 远程设备）
   - 查看控制台错误信息
   - 截图反馈给我

3. **字体太小/太大**
   - 这也是需要修复的问题
   - 记录具体情况

## 🛠️ 高级配置

### 修改端口号

如果 8080 端口被占用，可以修改为其他端口：

**Python 版本**：
编辑 `start-lan-server.bat`，找到最后一行：
```batch
python -m http.server 8080 --bind 0.0.0.0
```
将 `8080` 改为其他端口，如 `8081`

**Node.js 版本**：
编辑 `start-lan-server-node.bat`，找到最后两行：
```batch
start http://localhost:8080
http-server -p 8080 -a 0.0.0.0 -o
```
将两个 `8080` 都改为其他端口

---

### 查看本机 IP 地址

如果启动脚本没有正确显示 IP，可以手动查看：

**方法 1：命令行**
```bash
ipconfig
```
查找 "IPv4 地址" 或 Tailscale 的 IP（通常是 `100.x.x.x`）

**方法 2：PowerShell**
```powershell
Get-NetIPAddress | Where-Object {$_.AddressState -eq "Preferred"} | Select-Object IPAddress
```

**方法 3：Tailscale 应用**
- 打开 Tailscale
- 查看 "This device" 下的 IP 地址

---

### 删除防火墙规则

如果需要删除之前添加的防火墙规则：

```batch
netsh advfirewall firewall delete rule name="MatSci Calculator Lab - HTTP 8080"
```

或者：
1. 打开 `wf.msc`
2. 找到 "MatSci Calculator Lab - HTTP 8080" 规则
3. 右键点击 → 删除
