## 🔧 常见问题速查

### Q1: 自动化测试失败怎么办？
```
A: 
1. 查看失败的测试名称
2. 在 BUG_TRACKER.md 中创建条目
3. 根据错误信息定位问题
4. 优先修复🔴高优先级问题
```

### Q2: 如何判断问题是严重的？
```
A: 参考优先级分类：

🔴 高优先级（立即修复）:
  - 应用崩溃或白屏
  - 计算结果错误
  - 数据丢失
  - 主要功能不可用

🟡 中优先级（本周修复）:
  - UI布局错乱
  - 次要功能异常
  - 性能明显下降

🟢 低优先级（后续优化）:
  - 视觉细节
  - 文案优化
  - 边缘场景
```

### Q3: 测试过程中发现大量问题怎么办？
```
A: 
1. 不要慌张，这是正常的
2. 按优先级排序
3. 先修复🔴高优先级
4. 每天设定合理的修复目标
5. 记录经验教训避免重复
```

### Q4: 如何确保修复没有引入新问题？
```
A: 
1. 每次修复后运行自动化测试
2. 测试相关功能模块
3. 进行回归测试
4. 在不同浏览器中验证
```

### Q5: 什么时候可以开始封装移动应用？
```
A: 满足以下条件时：
  ✅ 达到"推荐发布标准"
  ✅ 所有🔴高优先级Bug已修复
  ✅ 跨浏览器测试通过
  ✅ 性能指标达标
  ✅ 添加了PWA支持（manifest + service worker）
```
