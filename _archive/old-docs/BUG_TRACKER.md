## 修复优先级指南

### 🔴 高优先级（立即修复）
- 应用崩溃或白屏
- 计算结果错误（影响科学性）
- 数据丢失
- 主要功能完全不可用
- 安全漏洞

### 🟡 中优先级（本周内修复）
- 次要功能异常
- UI布局错乱但不影响使用
- 性能明显下降
- 浏览器兼容性问题
- 用户体验不佳

### 🟢 低优先级（后续优化）
- 视觉细节调整
- 文案优化
- 边缘场景处理
- 性能微调
- 增强功能

## 测试验证流程

修复完成后，必须执行以下验证：

1. **单元测试验证**
   - [ ] 修复的问题不再复现
   - [ ] 相关功能正常工作
   - [ ] 边界条件正确处理

2. **回归测试**
   - [ ] 其他模块未受影响
   - [ ] 历史功能正常

3. **跨浏览器测试**
   - [ ] Chrome/Edge
   - [ ] Firefox
   - [ ] Safari

4. **响应式测试**
   - [ ] 桌面端 (>1280px)
   - [ ] 平板端 (820-1280px)
   - [ ] 手机端 (<820px)

5. **文档更新**
   - [ ] 更新此Bug日志
   - [ ] 如有必要，更新README
   - [ ] 记录学到的经验
